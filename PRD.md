# PRD: 고립청년 지원 정보 허브 (가칭)

- 작성일: 2026-07-02
- 작성자: koopy
- 상태: v1.0 (확정)
- 저장소명: `isolated-youth-news`

## 1. 배경 및 문제 정의

고립·은둔 청년을 지원하는 현장 종사자들은 아래 두 가지 정보 공백을 겪고 있다.

1. 고립청년 관련 정책, 통계, 사업 공모 등의 정보가 여러 기관 사이트에 흩어져 있어 파악에 시간이 든다.
2. 지원단체·기관들끼리 서로 어떤 사업을 하고 있는지, 어떤 소식이 있는지 알기 어려워 협업·중복 방지가 힘들다.

기존에 만든 [nonprofit-news](../nonprofit-news) 프로젝트가 비영리 지원사업 소식을 매일 자동 수집·게시하는 구조를 검증했으므로, 같은 패턴(GitHub Actions 자동 수집 + GitHub Pages 게시)을 고립청년 도메인에 맞게 재사용한다.

## 2. 목표

- 종사자가 매일 아침 접속 한 번으로 고립청년 관련 최신 정보와 타 기관 소식을 확인할 수 있게 한다.
- 별도 설치나 로그인 부담 없이 누구나 조회 가능한 공개 웹페이지를 GitHub 기반으로 무료 운영한다.
- 사람이 개입하지 않아도 매일 아침 지정된 조건으로 자동 갱신되는, 순수 조회용(read-only) 사이트로 운영한다.

## 3. 타겟 사용자

- **주 사용자**: 고립청년 지원단체·기관 종사자 (업무 중 정보 탐색 목적)
- **부차 사용자**: 고립청년 당사자 및 일반 대중 (조회만, 열람 전용)

## 4. 범위 (Scope)

### 4.1 In Scope (MVP)

| # | 기능 | 설명 |
|---|------|------|
| F1 | 좌측 섹션: 지정 단체·기업 소식 자동 수집 | 매일 09:00(KST) GitHub Actions가 지정 단체·기업명을 네이버·다음·구글에 검색해, 중복을 제거한 뒤 단체별 상위 5건을 메인 페이지 좌측에 게시 |
| F2 | 우측 섹션: 고립청년 공모·지원사업 소식 자동 수집 | 매일 09:00(KST) 사랑의열매·아름다운재단 등 주요 배분기관의 공모사업 게시판을 직접 크롤링해, 지금 신청 가능한 지원사업만 골라 메인 페이지 우측에 게시 (뉴스 기사는 다루지 않음) |
| F3 | 공개 웹 게시 | GitHub Pages로 정적 페이지 배포, 누구나 로그인 없이 조회 가능 (좌우 2단 레이아웃) |

수동 등록/업로드 기능 및 인증(비밀번호 등)은 두지 않는다. 사이트에 보이는 내용은 전부 자동 수집 결과로만 채워진다.

### 4.2 Out of Scope (1차 제외, 추후 검토)

- 수동 등록/업로드 기능 (인증 포함) — 자동 수집이 놓친 정보는 이번 범위에서 다루지 않음
- 당사자 대상 상담·매칭 기능
- 기관 간 실시간 채팅/메시징
- 통계 대시보드, 데이터 시각화
- 다국어 지원
- 모바일 앱 (반응형 웹으로 대응)

## 6. 수집 대상 소스 (확정)

### 6.1 좌측 — 지정 단체·기업 (고정 리스트, 총 7곳)

각 단체의 홈페이지를 직접 크롤링하지 않고, **단체명을 네이버·다음·구글 포털에 검색**해 나온 뉴스·소식 중 중복(URL/제목 기준)을 제거한 뒤 단체별 상위 5건만 게시한다. 이 방식 덕분에 공식 홈페이지가 없거나 확인이 안 된 단체도 이름만으로 수집 대상에 포함할 수 있다.

1. 사단법인 오늘은
2. 청년재단
3. 고민정거장 — 공식 홈페이지 없음(Instagram만 확인), 포털 검색 방식이라 이름만으로 수집 가능
4. (주)안무서운회사
5. 광명시 청년동
6. 무브유어마인드 — 홈페이지 URL 확신도 낮음, 포털 검색 방식이라 이름만으로 수집에는 영향 없음
7. 서울청년기지개센터

### 6.2 우측 — 고립청년 공모·지원사업 (실제 모집 중인 공모사업, 뉴스 기사 제외)

처음엔 포털 뉴스검색 위주로 설계했으나, "이미 끝난 일을 다루는 기사"만 모이고 "지금 신청 가능한 공모사업"은 잘 안 잡히는 문제가 있어 **주요 배분기관의 공모사업 게시판을 직접 크롤링하는 방식으로 전환**했다.

- **사랑의열매 온라인 배분신청 사이트** (`proposal.chest.or.kr`): 전국 지회의 공모사업 목록을 직접 크롤링. 이 게시판 자체가 마감 전 공모만 노출하는 구조라 별도 마감 여부 판단이 필요 없음
- **아름다운재단 배분신청 사이트** (`change.beautifulfund.org`): WordPress 검색 API로 "청년"/"고립"/"은둔" 키워드 검색
- **재단/모금회 홈페이지 (자동 도출)**: 구글 검색 기준으로 "고립청년"과 "재단" 두 키워드가 함께 검색되는 재단의 홈페이지를 수집 대상으로 선정. 사랑의열매·아름다운재단처럼 이미 알려진 곳 외에 새로 걸리는 재단도 자동으로 포함 — 단, Google Custom Search API가 결제 계정 미연동으로 현재 작동하지 않아 사실상 비어있는 상태. 대신 종사자가 직접 알려주는 재단 목록을 고정 리스트로 추가하는 방식을 병행하기로 함
- **관련 단체 홈페이지**: 고립청년 관련 활동을 하는 단체(좌측 7개 단체 포함)의 홈페이지 공고

이 4가지 모두 "고립"·"은둔"·"청년" 중 하나라도 언급된 것만 통과시켜, 사랑의열매·아름다운재단이 함께 다루는 아동·노인·장애인 등 다른 대상 사업과 구분한다.

**모집 중 / 마감 구분**: 결과발표류 공지나 제목에 박힌 마감일이 이미 지난 공고는 삭제하지 않고 "마감"으로 분류해 화면 하단에 별도로 보여준다 (사랑의열매 게시판은 애초에 마감 전 공모만 노출하는 구조라 항상 "모집 중"으로 분류).

## 9. 기술 방향 (초안)

모든 수집 방식은 **추가 비용이 발생하지 않는 무료 티어/방식만** 사용한다는 제약을 따른다.

- **수집 실행**: Node.js 스크래퍼(nonprofit-news의 `scrape.js` 구조 재사용) + GitHub Actions cron (매일 09:00 KST), 좌측/우측용 각각 실행. GitHub Actions 자체도 public 저장소 기준 무료.
- **포털 검색 공통 유틸(`scraper/search-utils.js`)**: 네이버·다음(카카오)·구글 검색을 좌/우 양쪽에서 재사용. 각 포털은 자체 무료 티어 API 키가 필요하며(네이버 검색 API, 카카오 검색 API, Google Custom Search API), 키가 없으면 해당 포털만 건너뛴다 — RSS 등 별도 폴백은 두지 않는다(과거 시도한 네이버 RSS 폴백은 더 이상 접근 가능한 URL이 아님을 확인).
- **좌측**: 단체명으로 네이버/다음/구글 검색 → 중복 제거 → 단체별 상위 5건. 홈페이지 크롤링 없음, API 불필요 항목 없음(키가 없으면 결과 0건).
  - 단체명이 일상 어휘와 겹치는 경우(예: "오늘은") 오탐이 나기 쉬워, 검색 결과를 최대치(네이버 100건/다음 50건)까지 넓게 가져온 뒤 "단체명 문구 포함 + 청년 언급"을 동시에 요구해 걸러낸다. 그래도 0건이면 "사단법인 오늘은"처럼 정식 명칭 전체 문구로 한 번 더 재시도.
- **우측** (뉴스 기사 제외, 직접 크롤링 위주):
  - **사랑의열매**: `proposal.chest.or.kr`의 공모사업 목록 페이지를 크롤링. 상세페이지 링크가 `javascript:fn_goDetail(...)` 형태라 실제 URL(`/popup/mainBusinessDetail.do?dstbBsnsCode=...`)을 코드에서 직접 조립. 목록 조회 시 `Referer` 헤더가 없으면 404가 나므로 반드시 포함
  - **아름다운재단**: `change.beautifulfund.org`의 WordPress REST API(`/wp-json/wp/v2/posts?search=...`)로 "청년"/"고립"/"은둔" 검색. "결과발표"류 문구, 제목에 박힌 마감일이 이미 지난 경우는 제외
  - 재단 후보 도출: Google Custom Search JSON API 무료 티어(하루 100건 쿼리 한도) 사용해 "고립청년 재단" 검색 → 결과 도메인을 재단 홈페이지 후보로 추출
    - **갱신 주기**: 매일 09:00에 다시 검색하지 않고, 별도 주기(예: 주 1회)로 재검색해 재단 후보 목록을 `docs/data/foundations.json` 등에 캐싱. 매일 09:00 콘텐츠 수집 시에는 이 캐싱된 목록 + 좌측 7개 단체 홈페이지(있는 경우)를 대상으로 크롤링. 무료 쿼리 한도를 여유 있게 유지하기 위한 목적.
    - **필터링**: 엄격한 검증 없이 검색 결과 상위 도메인을 그대로 후보에 포함하는 느슨한 방식으로 운영. 다소 부적합한 결과가 섞여도 허용하며, 이후 실제 운영하면서 눈에 띄는 오탐만 수동으로 제외 목록에 추가
  - 도출된 재단 홈페이지 + 관련 단체 홈페이지: 무료 크롤링 (공모/지원사업 키워드 포함된 게시물만 추출)
  - 우측은 좌측과 달리 결과를 누적하지 않고 매일 새로 수집한 것으로 교체 ("지금 모집 중"인 정보라 과거 결과를 쌓아둘 이유가 없음)
  - 유료 API 키, 유료 크롤링 서비스, 유료 서버(호스팅)는 사용하지 않음
- **저장**: `docs/data/left.json`, `docs/data/right.json`, `docs/data/foundations.json` 형태의 정적 데이터 파일
- **게시**: GitHub Pages (정적 HTML/JS, 무료), 좌우 2단 레이아웃, 조회 전용(쓰기 기능 없음)
